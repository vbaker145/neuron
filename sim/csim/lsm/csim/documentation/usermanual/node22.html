Global fields

There are a few fields which determine the global (i.e. not object specific) behavior of CSIM which can be modified via the set command: 


    » csim('set',<fieldname>,<value>);

Read/writeable fields

dt
The integration time step used by all objects

randSeed
The seed value for the random number generator

verboseLevel
How much stuff should be written to the console window (no effect yet!).

nThreads
Number of threads to use for simulation (no effect yet!). The multi-threaded version of CSIM is currently under development.

spikeOutput
Flag: 1 ... always output spikes as last cell element, 0 do not output spikes (obsolete).

The following fields are read-only:

t
The current virtual (simulation) time

step
The current time step; i.e. step=t/dt;
